# Speak2Draw-Agent-Studio

一个面向“AI 语音绘图工具”议题的项目。用户通过语音指令完成绘图创作，项目重点关注语音指令理解、绘图操作执行、容错处理、响应延迟和复杂指令拆解。

当前项目通过持续 commit、分支和 PR 的方式推进，避免最后一次性提交全部代码。

## 运行方式

```bash
npm install
npm run dev
```

浏览器打开终端中显示的本地地址。首次使用时，浏览器可能要求用户授予麦克风权限；这是浏览器平台限制。授权后，应用内绘图流程应通过语音完成。

如果页面提示麦克风权限被拒绝，请点击浏览器地址栏左侧的权限图标，将麦克风改为允许，然后刷新页面。建议使用 `http://127.0.0.1:5173/` 或 `localhost` 打开应用，不要使用普通局域网地址。

如果页面显示“正在监听”但语音指令不执行，可以先点击“测试麦克风”。这个测试只读取麦克风音量，不依赖浏览器语音识别服务：如果测试有声音输入，问题更可能在浏览器语音识别服务或网络；如果测试几乎没有输入，优先检查系统输入设备和麦克风音量。

为了避免“话还没说完就执行”，应用会等待短暂停顿后再提交语音指令。如果你看到“正在识别”提示，可以继续把完整指令说完，例如“画一个房子和太阳”。

## 可用语音示例

- 画一个红色圆形
- 画一个蓝色矩形
- 画一条黑色线
- 写文字你好
- 选择最后一个图形
- 把它改成黄色
- 把太阳改成红色
- 向右移动一点
- 删除太阳
- 把房子放到最上层
- 放大一点
- 撤销
- 重做
- 清空画布
- 导出图片
- 画一个房子和太阳
- 画一个红色房子和蓝色太阳，再把房子放到最上层
- 我能说什么
- 画布里有什么
- 当前选中的是什么

## 测试方式

```bash
npm test
npm run build
npm run test:e2e
```

端到端测试会启动本地 Vite 服务，并通过 `?e2e=1` 的测试入口模拟浏览器返回的语音文本，用来验证从语音文本到画布更新的完整链路。

## 开发规则

- 所有开发必须基于 Git 记录推进。
- 新功能必须通过独立分支和 PR 添加。
- 每个 PR 只做一件事。
- PR 描述必须包含标题、功能描述、实现思路、测试方式。
- main 分支必须始终保持可运行状态。
- 不要提交 `.env`、token、密钥、账号密码等敏感信息。
